## Policy
Active testing docs should be concise, current, and directly actionable. Large historical reports and generated books are searchable references only.
For user-facing runtime behavior, the default acceptance loop is user deployment followed by user-executed manual testing. AI-authored source-level tests and checks can support implementation confidence, but browser/runtime validation is authoritative only after the user confirms the change set is deployed to the target runtime(s).
