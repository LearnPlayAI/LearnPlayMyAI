# Completion Checklist

Use before closing any task.

- Governing docs were read first (`docs/aimem/aimem.md`).
- Root cause was identified (for bug/issue tasks).
- Tests/checks were run or any limitation was stated.
- Behavior docs were updated if behavior changed.
- `docs/func/README.md` post-change tasks were completed when applicable.
- `docs/handoverdocs/CHANGELOG.md` was updated for the change set.
- Final handover summary includes change summary and residual risks.
